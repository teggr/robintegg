Back in February I published [Java UI in 2026: The Complete Guide](/2026/02/08/java-ui-in-2026-the-complete-guide), an up-to-date overview of every UI option available to Java developers today. The response from the community was overwhelmingly positive — it clearly resonated with people who, like me, were frustrated at the lack of quality, current information on the topic.

Riding that wave of enthusiasm, I took things a step further and turned it into a dedicated website: [https://awesome-java-ui.com/](https://awesome-java-ui.com/).

![awesome java ui website]({{site.baseurl}}/images/awesome-java-ui.png)

## What is awesome-java-ui.com?

`awesome-java-ui` is a community-curated catalogue of Java UI frameworks and libraries. It covers the full spectrum of UI options for Java developers:

- **Web UI** — Vaadin, Wicket, TeaVM, HTMX integrations, j2html, PrimeFaces, Jakarta Faces, Thymeleaf, and more
- **Desktop UI** — JavaFX, Swing (with FlatLaf), NetBeans Platform, Eclipse RCP, JxBrowser, JCEF, SnapKit, Swing-Tree
- **Mobile UI** — Gluon Mobile, Codename One, Compose Multiplatform
- **Terminal UI** — JLine, Lanterna, Casciian, Latte, AsciiTable

Each entry includes a summary of the project, links to the official site and source code, screenshots, and code snippets so you can quickly get a feel for what the framework looks like in practice.

The goal is simple: give Java developers a single, trustworthy, up-to-date resource when they are evaluating which UI technology to use for their next project.

## How to contribute

The project is open source and contributions are very welcome. Whether you want to add a framework that's missing, improve an existing entry, fix outdated information, or suggest screenshots and code examples, here's how to get involved:

1. **Open an issue** on the GitHub repository to propose a new framework or report an inaccuracy
2. **Submit a pull request** with your changes — the repository uses GitHub issue templates and automated workflows to keep entries consistent, so there's a clear structure to follow
3. **Spread the word** — share the site with your team, post it in communities you are part of, or link to it from your own content

I've set things up so that the site runs largely autonomously: GitHub issue templates drive new entries through a consistent workflow and I review the output before merging. This means the barrier to contribution is low — you don't need to know the internals, just open an issue with the details and the workflow takes care of the rest.

You can find the project at: [https://github.com/teggr/awesome-java-ui](https://github.com/teggr/awesome-java-ui)
